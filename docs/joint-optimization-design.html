Neural nets as primitives: an incremental DAG-of-subnets architecture

A growing graph of small frozen neural modules, assembled by neural-guided search and gradient descent, selected by Minimum Description Length.

Status: design proposal for review — not yet implemented. Supersedes the earlier joint-from-scratch sketch.

Instead of a fixed library of symbolic primitives, the library becomes a DAG of small neural subnets grown one task at a time. To solve a task we pick a few attachment points on the existing graph, hang small new heads on them, freeze everything that came before, and train only the new heads by gradient descent until they reproduce the target. Several heads succeed; we keep the one that minimizes a Minimum Description Length objective. The graph ratchets toward the domain exactly as a DreamCoder library does — but its primitives are learned by gradient descent rather than mined by compression.

Lineage

This sits at the intersection of three known ideas, which is reassuring rather than novel-for-its-own-sake:

  • Cascade-Correlation (Fahlman & Lebiere, 1990) — add units one at a time, freeze all existing weights, train only the new unit against the residual. Every unit's activation becomes an input available to all future units.
  • Progressive Neural Networks (Rusu et al., 2016) — freeze old columns, attach a new column with lateral connections into the frozen features, train only the new params. No catastrophic forgetting by construction.
  • DreamCoder (Ellis et al., 2021) — a library that ratchets toward the domain, a wake/sleep loop, and an MDL objective where only novel structure costs description bits.

The objective: one incremental MDL functional

Every design choice below serves a single objective. For a task with examples $D$, solved by attaching a new head $h$ (architecture $A_h$, weights $\theta_h$) onto the frozen existing graph $G$, we minimize the description length in bits:

$$ L \;=\; \underbrace{L(A_h \mid G)}_{\text{which nodes to read + which block}} \;+\; \underbrace{L(\theta_h)}_{\text{new weights}} \;+\; \underbrace{L(D \mid G, A_h, \theta_h)}_{\text{data misfit (NLL)}} $$

The crucial property: the frozen graph $G$ costs zero marginal bits — it was already paid for by earlier tasks and is amortized across every task that reuses it. So per task we only code a small new head, and the marginal description length shrinks over time as more of the solution is reused. This is why "parameter count = program length" — which was meaningless for a net trained from scratch — becomes well-behaved here: a node is one unit regardless of weight magnitude, wiring is a discrete choice with a clean bit-cost, and only the small novel head enters the count.

Why selection is post-hoc, not a loss term

We do not fold the $L(A_h)+L(\theta_h)$ terms into the gradient objective during fitting. Two reasons: (1) the MDL penalty can dominate before any head has shown whether it can converge, biasing toward the shortest path prematurely; (2) once a head reaches zero data loss, no gradient flows through a selection gate, so a differentiable sparsity term has nothing to push against exactly when it matters. Instead we fit each candidate head with pure data loss, then select among the converged ones by MDL. Fitting and selection are cleanly separated.

Per-task growth: multi-head, freeze, fit, select

  1. Propose attachment points. The existing node-picker (the trained q-head) ranks nodes in $G$ as candidate inputs for a new head. Its per-node embedding already splits into a structural part and a value part — the node's activations on this task's examples — and the value part is exactly the right signal for "is this node useful for this target." Take a top-$K$ shortlist; this is what tames the combinatorial attachment space.
  2. Hang $K$ candidate heads. Each head reads one (or a few) shortlisted nodes and is a small stack of thin units topped by a decoder back to grid space.
  3. Freeze $G$, train the heads. All existing parameters are frozen — old tasks remain solved by construction. Each head trains by gradient descent on pure data loss until it reproduces the target (or is disqualified for failing to).
  4. Select by MDL. Among the heads that exactly fit, keep the one minimizing $L$ — fewest new bits, most reuse. The losing heads are discarded.
  5. Commit or discard. If the winning head's modules look reusable (see Library & pruning), commit them as permanent nodes in $G$; otherwise treat them as a throwaway solution that solved this task but does not enlarge the library.

Node granularity: everything is an output resolves hidden vs. output

There is no real distinction between "hidden features" and "outputs" — there is only a distinction if a node is a coarse sealed block. The moment every unit's output is a first-class node in the DAG, a later head can tap any earlier activation, and "the final answer of task 3" and "an intermediate feature inside task 3's subnet" are the same kind of thing: a node you can read from. Nothing valuable is discarded because nothing is hidden. This is cascade-correlation's topology, and it makes the neural DAG isomorphic to the symbolic one: a new unit reading from existing nodes is structurally identical to an App(f, a), which the picker already knows how to score.

We therefore separate two granularities that are easy to conflate:

  • Addressable node = a thin unit (linear + nonlinearity over the latent). Fine, so everything is reusable and nothing is sealed.
  • Per-task trained growth = a small head (a few thin units + a decoder), trained together. Every layer inside it still becomes an addressable node for future tasks.

Fine nodes also help the few-shot regime: a thin head has few parameters, so it generalizes better from ~3 examples than a fat block, and the residual problem it must solve — given rich frozen features — is much smaller than learning the task from pixels.

The uniform latent interface: the universal currency the enabler

For "any node can feed any node" to hold, edges need a common type. The uniform latent interface is the neural analog of the symbolic core's single Value type: in the untyped calculus any App(f, a) is admissible because everything on an edge is one universal type, and mismatches surface softly as Bottom at evaluation rather than as construction-time errors. The latent interface answers the same question for the neural DAG — what flows along the edges so that composition is always legal and any node is substitutable for any other?

Precisely which problems it solves

ProblemWithout a uniform interfaceWith it
Picker action space Action is "read node X through an adapter of shape S" — shape bookkeeping pollutes every action. Action is just "read node X" — a clean discrete symbol, exactly the App(f,a) shape the q-head scores.
Per-task MDL accounting Every edge carries a learned adapter with parameters; wiring is no longer free; the "small new head" balloons. Wiring is free; only the small head costs bits; the amortization story holds.
Attachment-ranking signal "Distance from node's activation to the target" isn't comparable across heterogeneous shapes. Every node's value lives in one space; usefulness is computed identically for all nodes.
Substitutability / reuse Substitutability is shape-gated; library-reuse semantics degrade. A node is characterized by its latent value and can stand in anywhere — hash-consing-like identity survives.

Honest caveat — it is a property-preservation choice, not a strict necessity. You could instead insert a learned linear projection at every edge and compose any shapes. The uniform latent is not solving an impossibility; it is preserving the four properties above that make the DAG behave like a program library rather than a tangle of shape-specific glue. It does not solve few-shot generalization or exactness.

"Uniform" means uniform in channels, not in total shape

ARC grids run from 1×1 to 30×30. If the latent were a single fixed-dimension vector, every variable-size grid would be forced through a fixed bottleneck — losing spatial information and actively fighting the domain. The right notion is a feature map of a fixed channel count $C$ at the grid's current $H\times W$: uniform in $C$ (so any node feeds any node channel-wise), variable in spatial extent (so the grid is not crushed). A fixed encoder maps the input grid into this space; per-task decoders map latents back to grids. This choice silently determines how much representation is expressible, and is the first thing to pin down in the next pass.

Wake / sleep: freeze-and-grow vs. consolidate-and-re-solve

Freezing buys no-forgetting and cheap training, but it ossifies: early subnets were trained to solve early tasks, not to be good reusable features, and left frozen forever the substrate becomes order-dependent — a bad early task poisons everything downstream. The corrective is a periodic joint-retrain stage, which is DreamCoder's consolidation / abstraction sleep. It is not optional polish; it is the load-bearing counterweight to freezing, and it has two requirements:

  • Replay buffer of all solved tasks. Consolidation jointly trains all parameters against a multi-task loss over the buffer; otherwise it reintroduces the forgetting that freezing avoided.
  • Re-solve early tasks. After consolidation, re-solve earlier tasks with the now-richer graph and keep the MDL-best solution. A task is not solved once; it is re-solved more cheaply as the library improves. This is what cancels order-dependence.

The rhythm:

WAKE freeze G, pick attachment points, train K heads, select by MDL, commit reusable modules SLEEP jointly retrain all params on the replay buffer → re-solve earlier tasks → keep MDL-best ───────────────────────────────────────────────────────────────────────────────────────────────── repeat: the graph ratchets toward the domain; per-task description length shrinks over time

Library & pruning

The graph would otherwise grow forever. Not every task's head should join the permanent DAG: commit a module only if it earns its MDL keep or gets reused (DreamCoder keeps only high-value abstractions). Otherwise the task is solved with a throwaway head that is discarded. Inference cost for any task is only the subgraph reachable from its head, not the whole graph, so a reuse-gated, pruned library keeps both memory and forward-pass cost bounded. Open: the exact criterion for "important enough to keep" vs. "throwaway."

The risk to keep in view

The deep reason symbolic search wins on ARC is that "simplest discrete program that fits 3 examples" is a near-magic prior, and "simplest head that fits 3 examples" is a continuous-smoothness prior, which is a worse match for ARC's compositional, exact, discrete rules. The incremental structure helps — composition shrinks each residual problem, and thin heads are well-regularized — but it does not fully close the gap. The MDL machinery makes the complexity measure principled; it does not guarantee the low-complexity attractor that gradient descent finds is the right rule.

Falsifiable first experiment. Take 5–10 ARC tasks the symbolic system already solves, hand-build the correct small head for each on top of a hand-chosen attachment, and check whether MDL-regularized fitting on 3 examples recovers the rule and generalizes to the test grid exactly. If the hand-fed, right-architecture case already fails to generalize from 3 examples, the search machinery is moot and the direction is dead — cheap to find out, and it isolates the one assumption everything rests on.

Open questions for the next pass

  • The latent: channel count $C$, how the fixed encoder/decoder boundary handles variable $H\times W$, and how much representation that makes expressible.
  • Head shape: depth/width of the per-task head, and the block pool the picker chooses from.
  • The keep-vs-throwaway criterion for committing modules to the permanent graph.
  • Consolidation schedule: how often to sleep, and how re-solving interacts with committed modules.
  • Whether the picker's job stays learnable as the node pool grows into the thousands.
